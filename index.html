Patterns
Six systemic failures identified.
01

Tasks marked "Done" without completion

VAs close tasks at 100% to meet KPI targets without completing work. Astrid (VA Lead) and Oriana confirmed on-record, March 18.

02

No visual QA before marking done

Wrong logos, placeholder content, old names go live. CEO Kai Stone told VAs: "you have to proof check the website."

03

Sites built ignoring client specs

Wrong service emphasis, wrong locations, content that doesn't match onboarding discussions.

04

Assets not uploaded, work never started

Client photos sit for months. Wrong stock images remain on live sites.

05

VA handoffs die silently

Camila to Oriana on Cross Country: marked Done, passed it, nobody finished it. Client lost.

06

Technical blockers not communicated

VAs hit bugs but don't report them. They skip work silently or mark it done.

Seven recommended changes.

Mandatory visual QA checklist before task closure

Verify logo, images, content, links. Screenshot of finished page attached to task notes.

Owner: Astrid Altuve / Kai Stone
P1

Fix CSP task photo attachment bug

Platform bug enables silent failures. Fix it so VAs have no excuse.

Owner: Michael Vanhoutte / Umar
P1

Review VA KPI structure

Owner: Astrid Altuve / Michael Vanhoutte
P2

Add "Blocked" task status in CSP

VAs have no way to flag blocked tasks. They mark Done or leave open.

Owner: Michael Vanhoutte
P2

Owner: Astrid Altuve
